+ В заголовке Backup/Restore ошибка: Rrestore вместо Restore

-/? При выборе каталога для backup выводить каталог, выбранный предыдущий раз.
У диалогов, в которых выбирается каталог для backup, заголовок должен быть
соответственно "Save database backup to", "Save shop backup to"

- После "Backup finished successfully" добавить ", press OK button to return to Control Panel".

-/? 1.2.1 Запоминать каталог в диалогах выбора файлоа
Заголовки диалогов соответственно
"Choose database backup file"
"Choose shop backup file"

1.2.2 Если не выбран SQL файл для restore, выводится "Error: SQL dump file does not exists". Не должно выводится вообще ничего.

Если не выбран SQL файл, кнопка "Restore databas" должна быть disabled.
Если не выбран SQL файл и|или TAR файл, кнопка "Restore shop" должна быть disabled.

1.2.3 После того, как обломился database restore,  должно выводится 

=========================================================
Restoring database, please, wait ... [FAILED]

Control Panel failed to restore shop database. Please, check that you have
selected valid SQL database backup file. Press "Cancel" button to continue.
=========================================================

После того, как база данных восстановлена успешно, должно выводится

=========================================================
Restoring database, please, wait ... [OK]

The shop database has been restored successfully. Press "OK" button to continue.
=========================================================

(Текстовки будут еще позднее согласованы с вербиком)



1.2.4 При Restore shop, если подсунуть в качества backup shop file лажу, 

Extracting files ...
Files sucessefuly extracted

Должно обругаться.


1.2.5 После success restore должно выводиться

...

The shop has been restored successfully. Press "OK" button to continue.
=========================================================

1.2.6 Если на сервере отсутствует каталог, куда должен идти restore, выдается 
"Connection failed"

Должен создаваться shop directory!
Если не удается создать каталок, выдавать соответствующее сообщение

=========================================================
Restoring shop, please, wait ... [FAILED]

Control Panel was unable to restore shop.

Possible reasons are:
1. You have specified an invalid FTP settings or shop URL
2. You have no permission to create shop directory on server. Please, contact
server support service for more details.

=========================================================

Текстовка, вожможно, будет изменена или расширена.
